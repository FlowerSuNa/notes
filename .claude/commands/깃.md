skeleton 파일만 선별하여 git 커밋 준비 상태를 점검하고, 사용자 확인 후 commit + push 합니다.
개인/업무 내용이 포함된 파일은 절대 커밋하지 않습니다.

## skeleton 기준

커밋 가능:
- `scripts/*.py`
- `.claude/commands/*.md`
- `CLAUDE.md`, `README.md`, `.gitignore`
- `*.template.*` 파일
- `*/.gitkeep`
- `inbox/README.md`

커밋 불가 (content):
- `routines.md`, `dashboard.html`
- `inbox/` 파일 (README 제외)
- `processed/` 파일
- `projects/**/meetings/`, `todos.md`, `specs/`
- `ideas/*.md`

## 실행 절차

### Step 1 — git status 확인
```bash
git status --short
git diff --stat HEAD
```

### Step 2 — 파일 분류
변경된 파일을 읽어 skeleton / content / unknown 으로 분류한다.

**unknown 파일 처리** (신규 파일이 skeleton인지 content인지 불분명할 때):
- 파일 경로와 내용을 확인하여 판단
- content 로 판단되면 `.gitignore` 에 추가 후 사용자에게 알림
- skeleton 으로 판단되면 커밋 대상에 포함

### Step 2.5 — 보안 검사
커밋 예정 skeleton 파일 전체를 읽어 아래 항목이 포함되어 있는지 확인한다.

**검사 항목:**
- 로컬 경로 (`C:\Users\`, `C:/Users`, `/home/`, `/Users/`)
- 사용자명 또는 숫자 ID가 포함된 경로 패턴
- 이메일 주소 (`@` 포함 문자열)
- API 키 / 토큰 패턴 (20자 이상 영숫자 연속, `sk-`, `token`, `key`, `secret`, `password` 포함 줄)
- 회사명, 프로젝트명, 도메인 등 업무 고유명사
- IP 주소 패턴

**의심 항목이 발견되면:**
```
🚨 보안 검사 경고

  파일: .claude/commands/xxx.md
  내용: (의심되는 줄 표시)
  이유: 로컬 경로 / 이메일 / API 키 등

커밋을 중단합니다. 해당 내용을 수정한 뒤 다시 실행해주세요.
```
→ 이후 단계 진행하지 않고 종료.

**이상 없으면:** "✅ 보안 검사 통과" 한 줄만 출력하고 Step 3으로 진행.

### Step 3 — 커밋 대상 미리보기 + 메시지 제안
변경된 skeleton 파일의 diff를 읽어 변경 내용을 분석한 뒤, 적합한 커밋 메시지를 직접 제안한다.

사용자에게 아래 형식으로 출력:

```
🔍 커밋 예정 파일 (skeleton only)

  M  .gitignore
  A  .claude/commands/루틴.md
  M  CLAUDE.md
  ...

⛔ 제외된 파일 (content — gitignore 적용)
  routines.md  →  .gitignore 에 포함됨
  ...

💬 제안 커밋 메시지:
  "<변경 내용을 분석한 메시지>"

이 메시지로 커밋할까요? 수정하려면 새 메시지를 입력해주세요. (또는 "취소"로 중단)
```

### Step 4 — 사용자 확인
- "응", "예", "ㅇㅇ" 등 긍정 응답이면 제안 메시지로 진행
- 새 메시지를 직접 입력하면 그 메시지로 진행
- "취소" 시 중단

### Step 5 — commit
```bash
git add <skeleton 파일들만>
git commit -m "<사용자 입력 메시지>"
```

### Step 6 — push 확인
```
✅ 커밋 완료: <메시지>

origin/main 으로 push 하시겠습니까? (예/아니오)
```
"예" 를 받으면:
```bash
git push
```

### Step 7 — 완료 보고
```
🚀 push 완료 — https://github.com/FlowerSuNa/notes
```

### Step 8 — Google Drive 동기화
push 성공 후 자동으로 실행한다.

```powershell
rclone sync . gdrive:notes --exclude ".git/**" --exclude ".claude/**"
```

완료 후 출력:
```
☁️ Google Drive 동기화 완료 — gdrive:notes
```

오류 발생 시:
```
⚠️ Drive 동기화 실패 — rclone 오류 메시지 표시
```
(push는 이미 완료된 상태이므로 Drive 실패가 전체를 취소하지 않음)
