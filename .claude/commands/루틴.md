사용자가 입력한 루틴 내용을 분석하여 routines.md 의 적절한 섹션에 깔끔하게 추가합니다.

사용법: /루틴 {내용}
예시:
  /루틴 매주 월요일 팀 스탠드업 회의 준비
  /루틴 매월 말일 비용 정산 확인
  /루틴 분기마다 모델 성능 리포트 작성

## 실행 절차

### Step 1 — 주기 파악
입력 내용에서 아래 주기 키워드를 감지한다:

| 감지 키워드 | 섹션 |
|------------|------|
| 매일, 날마다, daily | `## 매일` |
| 매주 월 / 화 / 수 / 목 / 금 | `## 매주 > ### 요일` |
| 매주 (요일 없음) | `## 매주 > ### 비정기` 대신 가장 적절한 요일 추정 또는 사용자에게 요일 질문 |
| 매월, 한달에 한번, 월말, 월초 | `## 매월` |
| 분기, 3개월마다 | `## 분기` |
| 키워드 없음 / 비정기 | `## 비정기` |

### Step 2 — 항목 문구 다듬기
- 동사로 끝나는 명확한 행동 문장으로 작성
- 불필요한 조사·어미 제거, 핵심 행동만 남김
- 대상(무엇을)과 행동(어떻게)이 모두 포함되도록
- 너무 길면 핵심만 남기고 세부 내용은 괄호로

좋은 예:
  - [ ] 서버 주요 코드 로컬 동기화 후 Bitbucket push
  - [ ] 팀 스탠드업 준비 (전날 작업 내용 정리)
  - [ ] 월말 비용 정산 확인 및 보고

나쁜 예:
  - [ ] 매주 금요일마다 서버에 있는 코드를 로컬로 옮기고 비트버킷에 푸쉬하는 것을 해야 함

### Step 3 — routines.md 에 추가
`routines.md` 를 읽고 해당 섹션 아래 마지막 항목 뒤에 추가한다.
섹션이 비어 있으면 헤더 바로 아래에 첫 항목으로 추가.

### Step 4 — 완료 보고
```
✅ 루틴 추가 완료

  섹션: ## 매주 > ### 금요일
  항목: - [ ] 서버 주요 코드 로컬 동기화 후 Bitbucket push
```
추가한 항목과 위치를 한 줄로 출력한다.
