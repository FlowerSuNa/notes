processed/ 에서 `상태: 대기중` 파일을 읽어 분류 제안을 보여주고,
사용자 확인 후 최종 위치에 저장합니다. 원본은 archive/ 로 이동합니다.

## 실행 절차

### Step 1 — 대기 파일 수집
`processed/` 디렉토리에서 내용에 `**상태**: 대기중` 이 포함된 `.md` 파일을 모두 읽는다.
없으면 "분류 대기 중인 파일이 없습니다. /정리 를 먼저 실행해주세요." 안내하고 종료.

### Step 2 — 분류 제안 테이블 출력
각 파일의 요약을 바탕으로 분류를 제안한다:

분류 옵션: `meeting` / `todo` / `idea` / `spec` / `skip`
- `meeting`: 회의, 미팅, 논의, 결정 내용이 있을 때
- `todo`: 할 일, 작업 목록, 액션 아이템이 주된 내용일 때
- `idea`: 아이디어, 구상, 기획 초안
- `spec`: 기능 명세, 요구사항, 설계 문서
- `skip`: 분류 불필요 또는 이미 처리됨

사용자에게 아래 형식으로 출력하고 수정을 요청한다:

```
📋 분류 대기 중 — N개 파일

| # | 파일 | 요약 | 제안 분류 | 프로젝트 |
|---|------|------|-----------|---------|
| 1 | 파일명.md | 요약 한 줄 | meeting | ? |
| 2 | 파일명.md | 요약 한 줄 | idea | — |
...

프로젝트명이 `?` 인 항목은 알려주세요.
수정할 내용이 있으면 "1번 todo로, 프로젝트는 한쌤" 형식으로 말씀해주세요.
없으면 "저장해줘"라고 하시면 됩니다.
```

### Step 3 — 사용자 확인/수정 대기
사용자가 수정 사항을 알려주면 테이블을 업데이트하고 재출력한다.
"저장해줘" 또는 "확인" 을 받으면 Step 4 진행.

### Step 4 — 최종 저장
확정된 분류에 따라:

**meeting:**
- `projects/<name>/meetings/` 디렉토리 생성 (없으면)
- `projects/<name>/meetings/YYYY-MM-DD.md` 생성 (회의록 템플릿 적용)
- 같은 날짜 파일이 이미 있으면 내용 추가(append)

**todo:**
- `projects/<name>/todos.md` 에 `- [ ] 항목` 형식으로 추가
- 파일 없으면 `# [프로젝트명] 할 일` 헤더와 함께 새로 생성

**idea:**
- `ideas/<slug>.md` 생성 (slug = 날짜_핵심키워드)

**spec:**
- `projects/<name>/specs/` 디렉토리 생성 (없으면)
- `projects/<name>/specs/<slug>.md` 생성

**skip:**
- 파일 이동 없음

### Step 5 — 아카이브 및 상태 업데이트
skip이 아닌 각 항목에 대해:
1. 원본 아카이브:
```bash
python scripts/archive_file.py <원본파일명>
```
2. processed/ 파일의 `**상태**: 대기중` 을 아래로 업데이트:
```
**상태**: 완료 → projects/xxx/meetings/YYYY-MM-DD.md
```

### Step 6 — 완료 보고
```
✅ 분류 완료

- meeting N건 → projects/...
- todo N건 → projects/...
- idea N건 → ideas/...
- skip N건

대시보드를 업데이트하려면 /대시보드 를 실행하세요.
```
