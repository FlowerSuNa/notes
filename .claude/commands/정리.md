inbox/ 파일을 스캔하여 **구조화된 분석 문서**로 가공한 뒤 processed/ 에 저장합니다.
분류는 하지 않습니다. 완료 후 /분류 실행을 안내합니다.

## 정리 품질 기준

단순 요약이 아니라 **내용을 재구조화**한다. 아래 기준을 따른다:

- 흩어진 메모를 **주제별 번호 섹션**으로 묶는다 (① ② ③ …)
- 각 섹션에서 **가설 / 결정 사항 / 액션 아이템 / 미결 사항**을 구분한다
- 원문의 `->` 화살표는 **실행 방향 또는 결론**으로 해석해 문장으로 풀어쓴다
- 수치(%, 기간, 건수)는 빠짐없이 보존한다
- 고유명사(시스템명, 프로젝트명, 담당자명)는 원문 그대로 유지한다
- 불명확한 주어("우리", "현업")는 문맥에서 추론 가능하면 구체화한다
- 분량에 제한 없음 — 원문이 길면 정리도 길어야 한다

## 실행 절차

### Step 1 — inbox 스캔
```bash
python scripts/process_inbox.py
```
`status: "empty"` 이면 "inbox/가 비어 있습니다." 안내하고 종료.

### Step 2 — 각 파일 재구조화
파일 내용을 읽고 아래 순서로 가공한다:

1. **주제 분류**: 원문에서 반복되거나 연결된 내용을 묶어 ①②③ 섹션 도출
2. **섹션별 서술**: 각 주제에 대해
   - 배경/현황 (왜 이 이슈가 나왔는가)
   - 분석/가설 (원인 추정, 검토 방향)
   - 결정 사항 (합의된 내용)
   - 액션 아이템 (누가, 무엇을 해야 하는가)
3. **미결 사항**: 답이 나오지 않은 질문, 확인이 필요한 항목
4. **키워드**: 3개 이내

### Step 3 — processed/ 저장
파일명: `processed/YYYY-MM-DD_<원본파일명 slug>.md`

형식:
```markdown
# [가공] 원본파일명

- **원본**: inbox/원본파일명.md
- **날짜**: YYYY-MM-DD
- **상태**: 대기중
- **키워드**: 키워드1, 키워드2, 키워드3

---

## 정리

### ① 섹션 제목
**배경**: …
**분석 / 가설**: …
**결정 사항**: …
**액션 아이템**:
- [ ] 내용 (담당: ?, 기한: ?)

### ② 섹션 제목
…

---

## 미결 사항
- 확인이 필요한 것들

---

## 원문
(원본 내용 그대로)
```

### Step 4 — 완료 보고
```
📥 정리 완료 — N개 파일이 processed/ 에 저장되었습니다.

1. 파일명.md — 섹션 ①②③ / 액션 N건
2. …

분류하려면 /분류 를 실행하세요.
```
