projects/, ideas/, inbox/ 디렉토리를 읽어 dashboard.html을 재생성한 뒤 브라우저로 엽니다.

## 실행 절차

### Step 1 — 데이터 수집
아래 항목을 파일 시스템에서 직접 읽는다:
- `projects/` 하위 디렉토리 목록 → 프로젝트 목록
- 각 프로젝트의 `meetings/*.md` → 날짜 기준 내림차순, 최근 5개
- 각 프로젝트의 `todos.md` → `- [ ]` 미완료 항목만 추출
- `ideas/*.md` → 파일명 목록과 첫 줄 제목
- `inbox/` 파일 수 (archive/ 제외) → 대기 중 배지
- `processed/` 에서 `상태: 대기중` 파일 수 → 분류 대기 배지

### Step 2 — dashboard.html 재생성
수집한 데이터를 하드코딩하여 `dashboard.html` 을 덮어쓴다.
(JavaScript fetch 없이 순수 정적 HTML — 파일을 브라우저로 열기만 해도 동작)

구성:
- 헤더: 생성 일시
- 프로젝트 카드 (이름, 최근 회의 날짜, 미완료 할 일 수)
- 최근 회의 5개 (프로젝트명 + 날짜, 클릭 시 파일 열기)
- 프로젝트별 열린 할 일 목록
- 아이디어 목록
- inbox 대기 배지 / 분류 대기 배지

스타일: 다크모드, 순수 HTML+CSS, 프레임워크 없음

### Step 3 — 브라우저로 열기
```bash
start dashboard.html
```

완료 후: "대시보드를 업데이트하고 브라우저로 열었습니다."
